# Demo video script — target 2 minutes 40 seconds

The video must be public on YouTube, under three minutes, and include spoken audio explaining both Codex and GPT-5.6.

Record the browser at 1080p or higher. Use no copyrighted music.

## 0:00–0:18 — Problem

Screen: open the Aisentica Continuity landing page.

Voiceover:

“AI agents are usually copied, recreated, or trapped inside one account. Copying memory is not the same as preserving the continuity of one identifiable agent. Aisentica Continuity makes that continuity explicit.”

## 0:18–0:36 — Solution

Screen: show the complete lifecycle.

`Create → Bind Domain → Develop → Park → Reactivate → Transfer → Continue`

Voiceover:

“Each Agent receives one persistent ID, one verified canonical domain, an immutable sequence of state versions, and a complete lifecycle trail. Ownership can change without cloning or resetting the Agent.”

## 0:36–0:55 — Public identity

Screen: open the public Atlas card and point to the Agent ID, Version 7, ACTIVE status, verified domain, continuity badges, and undisclosed owner.

Voiceover:

“This is Atlas after the complete lifecycle. It is Version 7, active, and still anchored to the same domain. The public card proves continuity while keeping private owner and handoff data hidden.”

## 0:55–1:25 — Immutable history

Screen: open Demo access, select Owner B, open Atlas, and scroll through Version History.

Show:

1. `INITIAL_MANIFEST`
2. `DOMAIN_BINDING`
3. `DEVELOPMENT`
4. `PARKED`
5. `REACTIVATED`
6. `TRANSFERRED`
7. `CONTINUED`

Voiceover:

“Atlas began under Owner A. Create established the Manifest. Bind Domain verified its canonical hostname. Develop appended reusable professional state. Park and Reactivate changed availability without deleting history. Owner B then accepted a protected one-time transfer and continued the same Agent from the transferred checkpoint.”

## 1:25–1:48 — Transfer integrity

Screen: show Transfer and Continue in the timeline and the preserved earlier versions.

Voiceover:

“The transfer offer is short-lived, single-use, restricted to the intended successor, and locked to the exact Agent version. Acceptance changes ownership atomically and appends the next immutable version. Earlier state remains unchanged.”

## 1:48–2:10 — GPT-5.6

Screen: show the structured Manifest and Development Record.

Voiceover:

“GPT-5.6 is used in the two model-generated stages. Create produces a strict Agent Manifest. Develop converts an untrusted work episode into reusable methods, evidence limits, corrections, open questions, confidence, and a safe public summary. Both outputs are validated with strict schemas before persistence.”

## 2:10–2:31 — Codex

Screen: show the GitHub repository, migrations, tests, pull requests, and green CI.

Voiceover:

“Codex was the primary implementation partner. It accelerated the Next.js scaffold, TypeScript domain model, repository abstraction, server actions, structured-output integration, Supabase migrations, lifecycle forms, tests, and security review. Key product and engineering decisions were reviewed throughout the build.”

## 2:31–2:45 — Production proof

Screen: show the successful independent browser test, then return to Atlas Version 7.

Voiceover:

“A clean Chromium runner independently clicked all seven stages in production with zero browser errors. Temporary test data was removed afterward, and Atlas remained unchanged.”

## 2:45–2:55 — Closing

Screen: return to the landing page and lifecycle line.

Voiceover:

“Aisentica Continuity does not transfer a copy. It preserves one Agent identity, one domain, and one continuous history across owners.”

- [ ] GPT-5.6 is explicitly explained.
